<!-- more -->

## 为什么选择 Hexo

在考虑了多种方案后，选择了 Hexo 的理由：

- **静态站点**：无需服务器，GitHub Pages 免费托管
- **Markdown 写作**：专注内容，无需关心 HTML
- **中文社区活跃**：遇到问题容易找到解答
- **主题丰富**：从简洁到绚丽，各种风格可选

## 环境准备

```bash
# 安装 Node.js（推荐 v18+）
# 访问 https://nodejs.org 下载

# 安装 Hexo CLI
npm install -g hexo-cli

# 验证安装
hexo -v
```

## 初始化项目

```bash
# 初始化博客
hexo init my-blog
cd my-blog

# 安装依赖
npm install

# 启动本地预览
hexo server
# 访问 http://localhost:4000
```

## 关键配置（`_config.yml`）

```yaml
title: 你的博客标题
author: 你的名字
url: https://你的用户名.github.io
language: zh-CN
timezone: Asia/Shanghai
```

## 部署到 GitHub Pages

本博客使用 **GitHub Actions** 自动化部署，每次推送到 `main` 分支时自动构建并发布。详见仓库根目录的 `README.md`。
